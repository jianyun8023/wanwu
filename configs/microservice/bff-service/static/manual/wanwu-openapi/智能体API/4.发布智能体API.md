# 发布智能体API

配置完成后，调用本接口发布智能体。发布后才能通过**智能体对话API**进行正式问答。

> **前提条件：** 发布前必须通过**配置更新API**完成以下配置，否则发布请求将报错：
> - **开场白（`prologue`）**：必须配置，不能为空
> - **大模型（`modelConfig`）**：必须配置有效的模型，不能为空
>
> 请参阅 [2.配置更新API.md](2.配置更新API.md) 完成配置后再调用本接口。

## 请求接口

| Type | Instructions |
| ---- | ---- |
| 方法 | Http |
| 请求URL | 按照应用实际API根地址，例如：`http://localhost:8081/service/api/openapi/v1/agent/publish` |
| 字符编码 | UTF-8 |
| 请求类型 | POST |
| 鉴权方式(header参数) | `Authorization: Bearer {API Key}` |
| 请求格式 | `Content-Type: application/json` |
| 响应格式 | `Content-Type: application/json` |

## 请求参数

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| assistantUuid | 是 | string | 智能体UUID |
| version | 是 | string | 发布版本号，格式为 `vX.Y.Z`，如 `v1.0.0`，且必须大于上一次发布的版本号 |
| publishType | 是 | string | 发布范围：`public`-系统公开，`organization`-组织内公开，`private`-私密 |
| desc | 是 | string | 本次发布的版本描述 |

## 响应参数

| Parameter | Type | Instructions |
| --------- | ---- | ---- |
| code | int | 状态码，0表示成功 |
| msg | string | 提示信息 |
| data | object | 成功时为 null |

## 调用示例

```shell
curl -k --location 'http://localhost:8081/service/api/openapi/v1/agent/publish' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "assistantUuid": "2004010127610351679",
    "version": "v1.0.0",
    "publishType": "private",
    "desc": "初始版本"
}'
```

## 响应示例

```json
{
    "code": 0,
    "msg": "",
    "data": null
}
```
