# 完整使用流程

> **典型流程**：新建智能体 → 更新配置 → 草稿态调试对话 → 发布 → 发布后正式对话。
> 需要多轮上下文时加「创建会话」，需要模型读取文件时加「上传文件」。

## 智能体查询

| 用途 | 接口 | 说明 |
| --- | --- | --- |
| 查询列表 | `GET /v1/agent/list` | 全部智能体（含草稿态与已发布态） |
| 按名称筛选 | `GET /v1/agent/list?name=关键字` | 名称模糊匹配 |
| 查询详情（草稿态） | `GET /v1/agent/info?uuid={uuid}` | 默认返回草稿态 |
| 查询详情（已发布态） | `GET /v1/agent/info?uuid={uuid}&published=true` | 返回已发布版本 |

> 返回的 `uuid` 是后续所有智能体接口的入参主键。

## 对话历史管理

### 已发布对话

| 操作 | 接口 | 说明 |
| --- | --- | --- |
| 对话列表 | `GET /v1/agent/conversation/list` | 按 uuid 分页 |
| 历史消息 | `GET /v1/agent/conversation/detail` | 按 conversation_id 分页 |
| 清空历史 | `DELETE /v1/agent/conversation/clear` | 保留 conversation_id，可继续问答 |
| 删除对话 | `DELETE /v1/agent/conversation` | 移除 conversation_id；传 detail_id 则只删该条消息 |

> **`clear` 还是 `delete`**：想保留同一对话继续问答用 `clear`；不再使用用 `delete`（下次问答前需重新创建对话）。

### 草稿态对话

> 草稿态每个智能体只有一条会话，全部接口通过 `uuid` 定位，无需 `conversation_id`。

| 操作 | 接口 | 说明 |
| --- | --- | --- |
| 草稿历史消息 | `GET /v1/agent/conversation/draft/detail` | 按 uuid 分页 |
| 删除草稿历史 | `DELETE /v1/agent/conversation/draft` | 传 detail_id 则只删该条消息 |

## 智能体生命周期管理

| 操作 | 接口 | 说明 |
| --- | --- | --- |
| 列出全部 | `GET /v1/agent/list` | 含草稿态与已发布态 |
| 删除智能体 | `DELETE /v1/agent` | 同时删除其已发布版本，**不可恢复** |
