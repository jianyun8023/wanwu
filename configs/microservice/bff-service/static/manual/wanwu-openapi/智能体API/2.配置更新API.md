# 智能体配置更新API

更新智能体的配置信息，包括提示词、模型配置、知识库配置等。

> **说明：** 本接口为**顶层字段级别覆盖**，未传入的顶层字段不会被修改，但传入的对象字段会整体覆盖。例如只更新提示词时，只传 `instructions` 即可，不必传其他字段；但传入 `knowledgeBaseConfig` 时，其中的知识库列表会完整替换当前配置。

## 请求接口

| Type | Instructions |
| ---- | ---- |
| 方法 | Http |
| 请求URL | 按照应用实际API根地址，例如：`http://localhost:8081/service/api/openapi/v1/agent/config` |
| 字符编码 | UTF-8 |
| 请求类型 | PUT |
| 鉴权方式(header参数) | `Authorization: Bearer {API Key}` |
| 请求格式 | `Content-Type: application/json` |
| 响应格式 | `Content-Type: application/json` |

## 顶层请求参数

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| assistantUuid | 是 | string | 智能体UUID |
| prologue | 否 | string | 开场白 |
| instructions | 否 | string | 系统提示词 |
| recommendQuestion | 否 | array\<string\> | 推荐问题列表，传空数组 `[]` 可清空 |
| modelConfig | 否 | object | 大模型配置，见下方说明 |
| knowledgeBaseConfig | 否 | object | 知识库配置，见下方说明 |
| rerankConfig | 否 | object | 知识库检索后重排序所用的 Rerank 模型配置，结构同 `modelConfig`，`modelType` 固定填 `rerank`，`modelId` 为空时清空 |
| visionConfig | 否 | object | 视觉配置，见下方说明 |
| memoryConfig | 否 | object | 对话记忆配置，见下方说明 |
| recommendConfig | 否 | object | 追问配置，见下方说明 |

## 响应参数

| Parameter | Type | Instructions |
| --------- | ---- | ---- |
| code | int | 状态码，0表示成功 |
| msg | string | 提示信息 |
| data | object | 成功时为 null |

## 响应示例

```json
{
    "code": 0,
    "msg": "",
    "data": null
}
```

## 配置项说明与示例

### 基础配置（开场白 / 提示词 / 推荐问题）

只传需要修改的字段，其他配置不受影响。

> **注意：** 如需使用**草稿态对话API**或**发布智能体API**，`prologue`（开场白）为必填项，不能为空。

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| prologue | 否 | string | 开场白 |
| instructions | 否 | string | 系统提示词 |
| recommendQuestion | 否 | array\<string\> | 推荐问题列表，传空数组 `[]` 可清空 |

**调用示例**

```shell
curl -k --location --request PUT 'http://localhost:8081/service/api/openapi/v1/agent/config' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "assistantUuid": "2004010127610351679",
    "prologue": "你好，我是翻译小助手",
    "instructions": "现在你是一个专业翻译人员，你的目标是把中文翻译成英文。",
    "recommendQuestion": [
        "把\"给我推荐三本书\"这句话翻译成英文"
    ]
}'
```

### 大模型配置（modelConfig）

> **注意：** 如需使用**草稿态对话API**或**发布智能体API**，必须配置有效的模型，否则请求将报错。

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| provider | 否 | string | 模型供应商标识，见下方 provider 对照表 |
| model | 否 | string | 模型标识名称，见下方字段来源说明 |
| modelId | 否 | string | 模型UUID，见下方字段来源说明；为空字符串 `""` 时清空模型配置 |
| modelType | 否 | string | 模型类型，对话模型固定填 `llm` |
| config | 否 | object | 模型推理参数，见下表 |

**字段来源说明**

以上字段均来源于平台**模型管理**页面，对应关系如下：

| 模型管理页面 | API 参数 | 示例值 |
| ---------- | -------- | ------ |
| 供应商 | `provider` | 联通元景 → `YuanJing` |
| 模型标识（模型详情页 -- 模型ID） | `model` | 例如：`deepseek-v3` |
| 模型UUID（模型详情页） | `modelId` | `<model-uuid>` |
| 模型类型（对话模型） | `modelType` | `llm` |

**provider 参数对照表**

| 平台显示名称 | `provider` 参数值 |
| ---------- | ---------------- |
| 联通元景 | `YuanJing` |
| OpenAI-API-compatible | `OpenAI-API-compatible` |
| DeepSeek | `DeepSeek` |
| 通义千问 | `Qwen` |
| Jina | `Jina` |
| 火山引擎 | `HuoShan` |
| 无问芯穹 | `Infini` |
| 百度千帆 | `QianFan` |
| 智谱 | `ZhiPu` |
| Ollama | `Ollama` |

#### config（模型推理参数）

> 不传 `config` 字段时，温度（temperature）、topP 等所有推理参数的默认值均为 **0**。如需自定义推理行为，请显式传入以下参数。

| Parameter | Type | Instructions |
| --------- | ---- | ---- |
| temperature | float | 生成温度，控制随机性，范围 0～2，建议值0.7 |
| temperatureEnable | bool | 是否启用 temperature 参数 |
| topP | float | 核采样概率阈值，范围 0～1，建议值1 |
| topPEnable | bool | 是否启用 topP 参数 |
| maxTokens | int | 最大输出 token 数，建议值512 |
| maxTokensEnable | bool | 是否启用 maxTokens 限制 |
| frequencyPenalty | float | 频率惩罚，降低重复词概率，范围 -2～2 |
| frequencyPenaltyEnable | bool | 是否启用 frequencyPenalty |
| presencePenalty | float | 存在惩罚，鼓励话题多样性，范围 -2～2 |
| presencePenaltyEnable | bool | 是否启用 presencePenalty |

**调用示例**

```shell
curl -k --location --request PUT 'http://localhost:8081/service/api/openapi/v1/agent/config' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "assistantUuid": "2004010127610351679",
    "modelConfig": {
        "provider": "<model-provider>",
        "model": "<model-id>",
        "modelId": "<model-uuid>",
        "modelType": "llm",
        "config": {
            "temperature": 0.7,
            "temperatureEnable": true,
            "topP": 1,
            "topPEnable": true,
            "maxTokens": 512,
            "maxTokensEnable": true,
            "frequencyPenalty": 0,
            "frequencyPenaltyEnable": true,
            "presencePenalty": 0,
            "presencePenaltyEnable": true
        }
    }
}'
```

> 清空模型配置：将 `modelConfig.modelId` 置为空字符串 `""`。

### 知识库配置（knowledgeBaseConfig）

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| knowledgebases | 否 | array | 关联的知识库列表，见下方说明；传空数组 `[]` 可解除全部关联 |
| config | 否 | object | 知识库检索参数，见下表；不传时默认使用**混合检索-自定义权重**模式 |

#### knowledgebases

> 知识库 ID 可在平台**知识库管理**页面获取。本字段为**全量替换**，传入的列表将完整覆盖当前已关联的知识库。

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| id | 是 | string | 知识库ID，在知识库管理页面获取 |

#### config（检索参数）

> 不传 `config` 时，默认使用**混合检索-自定义权重**模式。如需自定义检索行为，可按下表传参。

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| topK | 否 | int | 每次检索返回的知识片段数 |
| threshold | 否 | float | 相似度过滤阈值，低于此值的结果将被过滤 |
| maxHistory | 否 | int | 检索时携带的历史轮次数 |
| matchType | 否 | string | 检索类型：`vector`（向量检索）、`text`（文本检索）、`mix`（混合检索） |
| priorityMatch | 否 | int | 混合检索权重模式：1-自定义权重，0-自动 |
| semanticsPriority | 否 | float | 混合检索时语义（向量）权重，与 `keywordPriority` 之和为1 |
| keywordPriority | 否 | float | 混合检索时关键词权重，与 `semanticsPriority` 之和为1 |
| termWeight | 否 | float | 关键词系数 |
| termWeightEnable | 否 | bool | 是否启用关键词系数 |
| useGraph | 否 | bool | 是否启用知识图谱检索 |

**调用示例一：向量检索**

```shell
curl -k --location --request PUT 'http://localhost:8081/service/api/openapi/v1/agent/config' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "assistantUuid": "2004010127610351679",
    "knowledgeBaseConfig": {
        "knowledgebases": [
            { "id": "<knowledge-id>" }
        ],
        "config": {
            "topK": 5,
            "threshold": 0.4,
            "matchType": "vector"
        }
    }
}'
```

> 解除全部知识库关联：将 `knowledgebases` 传为空数组 `[]`。

**调用示例二：混合检索 + Rerank**

混合检索时需配置语义权重与关键词权重（两者之和为1），并可指定 Rerank 模型对结果重排序。

```shell
curl -k --location --request PUT 'http://localhost:8081/service/api/openapi/v1/agent/config' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "assistantUuid": "2004010127610351679",
    "knowledgeBaseConfig": {
        "knowledgebases": [
            { "id": "<knowledge-id-1>" },
            { "id": "<knowledge-id-2>" }
        ],
        "config": {
            "topK": 5,
            "threshold": 0.4,
            "matchType": "mix",
            "priorityMatch": 1,
            "semanticsPriority": 0.2,
            "keywordPriority": 0.8,
            "termWeight": 1,
            "termWeightEnable": true
        }
    },
    "rerankConfig": {
        "provider": "<model-provider>",
        "model": "<model-id>",
        "modelId": "<rerank-model-uuid>",
        "modelType": "rerank"
    }
}'
```

### 视觉配置（visionConfig）

> **前提条件：** 仅当 `modelConfig` 中选择的模型支持**图文问答**时，视觉配置才会生效。

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| picNum | 否 | int | 单次对话最多携带的图片数量，0 表示不限制 |

### 对话记忆配置（memoryConfig）

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| maxHistoryLength | 否 | int | 最大历史会话轮次，超出后滚动丢弃最早的轮次 |

### 追问配置（recommendConfig）

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| recommendEnable | 否 | bool | 追问功能总开关 |
| modelConfig | 否 | object | 追问所用模型配置，结构同 `modelConfig` |
| maxHistory | 否 | int | 追问时携带的最大历史轮次 |

**调用示例**

```shell
curl -k --location --request PUT 'http://localhost:8081/service/api/openapi/v1/agent/config' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "assistantUuid": "2004010127610351679",
    "recommendConfig": {
        "recommendEnable": true,
        "modelConfig": {
            "provider": "<model-provider>",
            "model": "<model-id>",
            "modelId": "<model-uuid>",
            "modelType": "llm",
            "config": {
                "temperature": 0.7,
                "temperatureEnable": true,
                "topP": 1,
                "topPEnable": true,
                "maxTokens": 512,
                "maxTokensEnable": true,
                "frequencyPenalty": 0,
                "frequencyPenaltyEnable": true,
                "presencePenalty": 0,
                "presencePenaltyEnable": true
            }
        },
        "maxHistory": 5
    }
}'
```
