# 完整使用流程

## 查询智能体列表

```
GET /v1/agent/list                          # 全部智能体（含草稿态与已发布态）
GET /v1/agent/list?name=关键字               # 按名称模糊筛选
```

返回的 `uuid` 是后续所有智能体接口的入参主键。

## 查询智能体详情

```
GET /v1/agent/info?uuid={uuid}              # 草稿态（默认）
GET /v1/agent/info?uuid={uuid}&published=true  # 已发布态
```

## 正式对话流程

```
第一步（可选）：新建智能体
POST /v1/agent
获取响应中的 uuid
         ↓
第二步：配置智能体（如已通过平台配置可跳过）
PUT /v1/agent/config
传入 assistantUuid 及需要更新的配置项
注意：发布前必须配置 prologue（开场白）和 modelConfig（大模型），否则发布将报错
         ↓
第三步：发布智能体（正式对话必须）
POST /v1/agent/publish
传入 assistantUuid、version、publishType
         ↓
第四步（可选）：创建会话（需要多轮对话时才需要）
POST /v1/agent/conversation
传入 uuid，获取响应中的 conversation_id；不传时为单轮对话
         ↓
第五步（可选）：上传文件（仅需携带文件时）
POST /v1/file/upload/direct
获取响应中的 fileId  → 填入 file_info[].fileName
            filePath → 填入 file_info[].fileUrl
            fileSize → 填入 file_info[].fileSize
         ↓
第六步：发起对话
POST /v1/agent/chat
携带 uuid 和 conversation_id，如有文件则同时携带 file_info
```

## 草稿态调试流程（无需发布）

```
第一步（可选）：新建智能体
POST /v1/agent
         ↓
第二步：配置智能体（如已通过平台配置可跳过）
PUT /v1/agent/config
注意：草稿态对话前必须配置 prologue（开场白）和 modelConfig（大模型），否则对话将报错
         ↓
第三步（可选）：创建会话（需要多轮对话时才需要）
POST /v1/agent/conversation
获取 conversation_id，后续对话中携带可保留上下文
         ↓
第四步：草稿态对话（不计入统计）
POST /v1/agent/chat/draft
conversation_id 可不传，不传时为单轮对话
```

## 对话历史管理

### 已发布对话

```
GET    /v1/agent/conversation/list                # 对话列表（按 uuid 分页）
GET    /v1/agent/conversation/detail              # 对话历史消息（按 conversation_id 分页）
DELETE /v1/agent/conversation                     # 删除对话；传 detail_id 则只删该条消息
DELETE /v1/agent/conversation/clear               # 清空历史但保留对话 ID
```

- 仅清空消息后还想继续基于同一对话问答 → 用 `clear`
- 不再使用该对话 → 用 `delete`，下次问答前需重新创建对话

### 草稿态对话

草稿态每个智能体只有一条会话，全部接口通过 `uuid` 定位，无需 `conversation_id`。

```
GET    /v1/agent/conversation/draft/detail        # 草稿历史消息（按 uuid 分页）
DELETE /v1/agent/conversation/draft               # 删除草稿历史；传 detail_id 则只删该条消息
```
