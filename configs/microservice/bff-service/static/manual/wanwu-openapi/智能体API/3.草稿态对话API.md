# 草稿态对话API

无需发布，直接基于当前草稿配置进行问答，适用于调试和测试阶段。对话不计入统计数据。

> **前提条件：** 调用草稿态对话前，必须通过**配置更新API**完成以下配置，否则对话将报错：
> - **开场白（`prologue`）**：必须配置，不能为空
> - **大模型（`modelConfig`）**：必须配置有效的模型，不能为空
>
> 请参阅 [2.配置更新API.md](2.配置更新API.md) 完成配置后再调用本接口。

> **与智能体对话API的区别：** 草稿态对话使用当前未发布的最新配置，而智能体对话API使用最近一次已发布的版本。

## 请求接口

| Type | Instructions |
| ---- | ---- |
| 方法 | Http |
| 请求URL | 按照应用实际API根地址，例如：`http://localhost:8081/service/api/openapi/v1/agent/chat/draft` |
| 字符编码 | UTF-8 |
| 请求类型 | POST |
| 鉴权方式(header参数) | `Authorization: Bearer {API Key}` |
| 请求格式 | `Content-Type: application/json` |
| 响应格式 | `Content-Type: text/event-stream`（SSE 流式） |

## 请求参数

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| uuid | 是 | string | 智能体UUID |
| conversation_id | 否 | string | 会话ID，由创建对话接口返回；不传时为单轮对话，无历史上下文 |
| query | 是 | string | 用户输入的问题或指令 |
| file_info | 否 | array | 随对话携带的文件列表，结构同智能体对话API。**当前一次仅支持 1 个文件**，传入超过 1 个会返回参数错误 |

## 响应参数

响应结构与**智能体对话API**完全相同，参见 [6.智能体对话API.md](6.智能体对话API.md)。

## 调用示例

### 单轮（不携带 conversation_id）

```shell
curl -k --no-buffer --location 'http://localhost:8081/service/api/openapi/v1/agent/chat/draft' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "uuid": "2004010127610351679",
    "query": "你好，请介绍一下你自己"
}'
```

### 多轮（携带 conversation_id）

```shell
curl -k --no-buffer --location 'http://localhost:8081/service/api/openapi/v1/agent/chat/draft' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "uuid": "2004010127610351679",
    "conversation_id": "56",
    "query": "你好，请介绍一下你自己"
}'
```
