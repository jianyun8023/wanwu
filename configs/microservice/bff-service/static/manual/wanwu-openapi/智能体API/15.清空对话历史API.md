# 清空对话历史API（含按条删消息）

清理指定**已发布对话**的消息（ES 数据），**保留对话 ID 本身**，可以继续基于该对话 ID 发起新的问答。
- 不传 `detail_id`：清空整个对话的全部消息
- 传入 `detail_id`：仅删除该条消息

> 与 [14.删除对话API.md](14.删除对话API.md) 的区别：删除对话会同时移除 conversation_id，再次问答需先调用[创建对话API](5.创建对话API.md)；本接口只清消息内容，conversation_id 仍可复用。

## 请求接口

| Type | Instructions |
| ---- | ---- |
| 方法 | Http |
| 请求URL | 按照应用实际API根地址，例如：`http://localhost:8081/service/api/openapi/v1/agent/conversation/clear` |
| 字符编码 | UTF-8 |
| 请求类型 | DELETE |
| 鉴权方式(header参数) | `Authorization: Bearer {API Key}` |
| 请求格式 | 参数走 query string，无请求体 |
| 响应格式 | `Content-Type: application/json` |

## 请求参数（query）

| Parameter | Required | Type | Instructions |
| --------- | -------- | ---- | ---- |
| conversation_id | 是 | string | 对话ID |
| detail_id | 否 | string | 消息ID；不传时清空整个对话的全部消息，传入时仅删除该条消息 |

## 响应参数

| Parameter | Type | Instructions |
| --------- | ---- | ---- |
| code | int | 状态码，0表示成功 |
| msg | string | 提示信息 |
| data | object | 空对象 |

## 调用示例

### 清空整个对话的全部消息

```shell
curl -k --location --request DELETE 'http://localhost:8081/service/api/openapi/v1/agent/conversation/clear?conversation_id=56' \
--header 'Authorization: Bearer <API Key>'
```

### 仅删除某一条消息

```shell
curl -k --location --request DELETE 'http://localhost:8081/service/api/openapi/v1/agent/conversation/clear?conversation_id=56&detail_id=1001' \
--header 'Authorization: Bearer <API Key>'
```

## 响应示例

```json
{
    "code": 0,
    "msg": "",
    "data": null
}
```
