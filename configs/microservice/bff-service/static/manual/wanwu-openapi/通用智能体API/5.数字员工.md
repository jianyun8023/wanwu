# 数字员工（DIP Agent）

> **@ 格式**：`@资源名称`。支持中文、英文、数字、下划线（`_`）、连字符（`-`）。

---

## 1. 数字员工（DIP Agent）

数字员工是一种特殊的知识网络智能体，每个员工拥有独立的任务、技能和知识体系。使用数字员工时：

- **必须** `agentId` 设为 `"DIP Agent"`
- **必须** @ 一个数字员工（只能一个）
- @ 格式：`@员工名称 用户消息`

### 使用方式

在消息开头以 `@员工名称` 指定数字员工，后跟具体的任务指令。**必须 @ 一个数字员工，不支持省略**：

```
@知识网络构建专员 帮我查询知识网络中的实体关系
```

```
@知识网络构建专员 分析数据表结构并生成实体映射
```

### 调用示例

```shell
curl -k --no-buffer --location 'http://localhost:8081/service/api/openapi/v1/wga/conversation/chat' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <API Key>' \
--data '{
    "threadId": "dbb35657-1ca7-4da9-9056-5b51fda085d1",
    "modelUuid": "2030935982164938752",
    "agentId": "DIP Agent",
    "messages": [
        {
            "id": "msg-1",
            "role": "user",
            "content": "@知识网络构建专员 帮我查询知识网络中的实体关系"
        }
    ]
}'
```

> **注意：**
> - **必须**在消息开头 @ 一个数字员工，否则无法指定执行者。
> - 只能 @ 一个数字员工。
> - `@员工名称` 和后面的消息内容之间用**空格**隔开。


---

## 2. 对比总结

| 维度 | 数字员工（DIP Agent） | @ 工具（通用智能体） |
|------|---------------------|---------------------|
| agentId | 必须为 `DIP Agent` | Supervisor 或其他子智能体 |
| @ 数量 | **仅一个**数字员工 | **支持多个**，空格分隔 |
| @ 位置 | 消息**开头** | 消息任意位置 |
| 用途 | 指定执行任务的数字员工 | 加载对话所需的工具/知识/技能 |
| 解析方式 | 取 @ 后到第一个空格为止 | 正则全局匹配所有 `@名称` |

> **提示：** 数字员工模式下也可以同时 @ 工具，@ 工具会被识别加载（通过非 DIP 模式的 ontology 处理），但不影响数字员工的指定规则。
