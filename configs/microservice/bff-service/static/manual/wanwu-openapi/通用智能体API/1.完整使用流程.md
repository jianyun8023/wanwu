## @ 工具（在消息中指定资源）

Supervisor 和子智能体支持在消息中 `@资源名称` 来加载特定工具、知识库、工作流等。@ 格式支持中文、英文、数字、下划线、连字符。

### 支持的资源类型

| 资源类型 | 说明 | 示例 |
|---------|------|------|
| MCP | MCP 工具服务 | `@MCP名称` |
| 工作流（workflow） | 预定义工作流 | `@工作流名称` |
| 技能（skill） | AI 技能 | `@技能名称` |
| 智能体（assistant） | 智能体 | `@智能体名称` |
| 知识库（knowledge） | 知识检索库 | `@知识库名称` |
| 知识网络（ontology） | 本体知识网络 | `@知识网络名称` |

### 使用示例

**@ 单个工具：**

```
@必应搜索中文 搜索元景万悟智能体平台的最新动态
```

**@ 多个工具（空格分隔）：**

```
@必应搜索中文 @企业知识库 请介绍元景万悟智能体平台的核心功能和架构
```

```
@必应搜索中文 @高德地图 请分析元景万悟智能体平台在北京地区的应用场景
```

> **@ 工具规则：**
> - 多个 @ 工具之间用**空格**隔开，工具和消息正文之间也用**空格**隔开。
> - 工具名称需与平台中配置的**资源名称**完全一致，否则无法识别。
> - 不同的资源类型共享同一套 @ 语法，系统会根据资源名称自动搜索匹配的资源。

## 对话历史管理

| 操作 | 接口 | 说明 |
| --- | --- | --- |
| 对话列表 | `GET /v1/wga/conversation/list` | 分页查询当前用户的对话列表 |
| 对话历史 | `GET /v1/wga/conversation/detail` | 按 threadId 查看某次对话的 AG-UI 事件流历史 |
| 删除对话 | `DELETE /v1/wga/conversation` | 删除指定对话及其全部历史记录 |

## 子智能体说明

WGA 采用 **Supervisor（监督者）+ 子智能体** 架构。用户通过 `POST /v1/wga/conversation/chat` 的 `agentId` 参数指定与哪个智能体对话：

| agentId | 名称 | 说明 |
|---------|------|------|
| 不传或留空 | Supervisor Agent | **默认入口**，自动理解用户意图并路由到合适的子智能体 |
| `Deep Research Agent` | 深度研究 | 多阶段深度调研（计划→研究→报告） |
| `Data Analysis Agent` | 数据分析 | 数据分析智能体，支持上传 Excel/CSV 文件进行分析 |
| `PPT Agent` | 创建PPT | PPT 文件生成智能体 |
| `Excel Agent` | 创建EXCEL | Excel 文件生成智能体 |
| `Html Agent` | 创建网页 | HTML 网页生成智能体 |

> **注意：** 以 "Skill" 开头的内部智能体（如 Skill Chat Agent）不对外开放，传入会返回参数错误。

## SSE 事件类型说明

`POST /v1/wga/conversation/chat` 返回 SSE 事件流，主要事件类型如下：

| 事件类型 | 说明 |
|---------|------|
| `RUN_STARTED` | 运行开始，返回 threadId、runId |
| `RUN_FINISHED` | 运行正常结束 |
| `RUN_ERROR` | 运行出错，返回错误信息 |
| `TEXT_MESSAGE_START / CONTENT / END` | 文本消息流（role=assistant 为 Agent 回复） |
| `REASONING_MESSAGE_START / CONTENT / END` | 推理/思考过程 |
| `TOOL_CALL_START / ARGS / END / RESULT` | 工具调用及结果 |
| `ACTIVITY_SNAPSHOT` | 活动快照，含子智能体切换、工作区更新等 |

### ACTIVITY_SNAPSHOT 的 activityType

| activityType | 说明 |
|-------------|------|
| `sub_agent` | 子智能体执行状态变化 |
| `workspace` | 工作区文件更新 |
| `question` | 智能体提问（需人工介入） |
