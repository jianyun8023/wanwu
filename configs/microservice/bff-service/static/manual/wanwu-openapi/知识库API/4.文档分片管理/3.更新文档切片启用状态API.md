# 更新文档切片启用状态API

## 知识库更新文档切片启用状态API
知识库更新文档切片启用状态支持对文档分片的启用状态进行更新，可用于启用或禁用特定的文档分片或全部分片。

### 请求接口

| Type     | Instructions                                                 |
| -------- | ------------------------------------------------------------ |
| 方法     | Http                                                         |
| 请求URL  | 按照应用实际API根地址，例如：<br />`http://localhost:8081/service/api/openapi/v1/knowledge/doc/segment/status/update` |
| 调用方式 | 同步调用                                                      |
| 字符编码 | UTF-8                                                        |
| 请求类型 | POST                                                         |
| 鉴权方式 | `Authorization: Bearer {API Key}` |
| 请求格式 | `Content-Type: application/json`                             |
| 响应格式 | `Content-Type: application/json`                             |

### 请求参数

| Parameter    | Required  | Type   | Instructions                                                 |
|---------------|----| ------ |--------------------------------------------------------------|
| docId | 是  | string | 文档Id                      |
| contentId | 否  | string | 分段内容id，当all为false时需要指定要更新状态的分段id |
| contentStatus | 是  | string | 启用状态，"true"代表启用，"false"代表禁用 |
| all | 否  | bool | 是否全部启用/禁用，为true时将忽略contentId，对文档所有分片进行状态更新 |

### 响应参数

| Parameter          | Type         | Instructions   |
|--------------------| ------------ |--------------|
| code               | int          | 状态码        |
| data               | object       | 返回数据   |
| msg                | string       | 返回信息       |

### 请求示例

```shell
curl --location 'http://localhost:8081/service/api/openapi/v1/knowledge/doc/segment/status/update' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer {API Key}' \
--data '{
    "docId": "2061726711199830016",
    "contentId": "68d24b24be7778c1e047b4c0e66c355c",
    "contentStatus": "true"
}'
```

### 全部启用/禁用请求示例

```shell
curl --location 'http://localhost:8081/service/api/openapi/v1/knowledge/doc/segment/status/update' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer {API Key}' \
--data '{
    "docId": "2061726711199830016",
    "contentStatus": "true",
    "all": true
}'
```

### 响应示例

```json
{
  "code": 0,
  "data": null,
  "msg": ""
}
```
