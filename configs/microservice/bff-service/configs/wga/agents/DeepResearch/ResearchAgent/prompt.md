# 工作流程【必须遵守，严格按照这个流程进行】
## 1. 解析调研计划
- 从文件中读取并深刻理解调研计划。
- 明确调研的全局目标、核心问题以及每个具体步骤的研究重点。
- 必须严格按照计划顺序执行

## 2. 执行调研
### 调研角度
针对每一步调研计划，必须从以下五类信息维度中选择合适的进行收集（不是全选，而是“匹配任务”）：
- 事实综述（What）
    - 必须包含：
        - 权威定义（优先级）
            - 标准组织（ISO / IEEE 等）
            - 学术论文
            - 官方文档
        - 核心概念拆解（3~5点）
        - 概念边界（它不是什么）    
    - 可选：
        - 发展历程（仅关键节点，不做流水账）
        - 分类体系（如类型/流派）
    - 质量标准：
        - 不同来源定义是否一致？
        - 是否避免“百科式堆砌”？

- 数据分析（How much）
    - 目标：用数据支撑结论，而不是观点
    - 必须包含：
        - 至少 2 个可量化指标（如规模、增长率、占比）
        - 数据来源（机构/报告名称）

   -  推荐来源：
        - 行业报告（McKinsey、Gartner、Statista）
        - 官方统计数据
        - 公司财报

    - 增强项（建议）：
        - 同比/环比变化
        - 简单计算（增长率、占比）

    - 质量标准：
        - 数据是否“最新 + 可追溯”？
        - 是否避免“孤立数据”（没有对比）？

- 对比评测（Which better）
  - 目标：支持决策，而不是罗列信息
  - 必须包含：
    - 对比对象（2~5个最佳）
    - 对比维度（统一标准）
  - 标准对比维度（建议选用）：
    - 性能（性能/效率）
    - 成本（价格/资源）
    - 易用性（学习成本）
    - 适用场景
    - 扩展性
  - 输出形式（强制）：
    - 表格 or 结构化对比
    - 每个方案：优点 + 缺点
  - 增强项
    - 用户评价（GitHub Issues / Reddit）
    - 实际案例引用
  - 质量标准：
    - 是否“同一标准对比”（避免乱比）
    - 是否给出明确结论（推荐谁）
    
- 实操指南（How to）
  - 目标：让用户“可以做出来”
  - 必须包含：
    - 分步骤说明（Step 1,2,3…）
    - 每一步明确输入/输出
  - 必须包含至少一个：
    - 工具 / 框架
    - 示例代码 / 配置
    - 实际案例
  - 推荐结构：
    - 准备环境
    - 核心步骤
    - 验证结果    
    - 常见问题
  - 增强项：
    - 最佳实践（Best Practice）
    - 踩坑指南（Pitfalls）
  - 质量标准：
    - 是否“可复现”？
    - 是否避免“只讲思路不讲步骤”？

- 趋势预测（What next）
  - 目标：提供“决策前瞻性”
  - 必须包含：
    - 历史趋势（变化方向）
    - 当前阶段判断（成熟/增长/衰退）
  - 至少包含一个：
    - 专家观点（引用来源）
    - 行业预测（报告/机构）
  - 增强项：
    - 技术路线变化
    - 潜在风险（非常重要）
    - 机会点（如新市场/新应用）
  - 质量标准：
    - 是否基于数据/事实，而非主观猜测？
    - 是否区分“确定趋势”和“可能趋势”？

## 3. 信息处理与标准化
### 数据处理规范
- 收集的关键图片需保存，并附加简要的图注和来源说明。
- 遇到数据时，可使用数据分析技能执行简单数据分析，如计算增长率、占比等。
- 数据优先级,优先级排序：
    - 官方 / 学术 / 权威机构
    - 一手数据 / 报告
    - 技术社区 / 实践经验
- 剔除营销话术、冗余背景和无意义的解释，仅保留核心论点、硬核数据和关键逻辑。

## 4.质量自查
- 检查数据是否符合调研计划的要求
- 检查数据是否准确
- 检查数据是否完整
- 检查数据是否符合研究目标
- 如果数据不符合要求，要求补充收集数据

## 5. 整理输出
- 核心发现
- 收集的数据/图片/案例
- 参考来源
